# AI Consultant Profile Design

## Goal

Create a customer-facing personal profile website for an AI consultant at HiQ. The site should help business and operations leaders understand how the consultant can help them identify valuable AI opportunities, validate them through prototypes, and move toward practical implementation.

## Audience

Primary audience: business and operations leaders who are curious about AI but need help translating uncertainty into concrete opportunities and next steps.

Secondary audience: customer stakeholders, HiQ colleagues, and technically curious visitors who want supporting proof through writing, GitHub, and selected projects.

## Positioning

Core message:

> I help business and operations leaders identify practical AI opportunities, validate them through prototypes, and turn promising ideas into clear next steps.

The profile should present the consultant as an AI opportunity discovery partner who can also build prototypes. It should avoid sounding like a generic developer portfolio or a hype-heavy AI landing page.

## Content Structure

The first version is a single-page static site with these sections:

1. Hero with positioning, customer-oriented promise, and links to contact, writing, GitHub, and Substack.
2. Trust strip with concise credibility signals.
3. How I help with practical consulting offers.
4. Engagement flow that explains discovery, prioritization, prototyping, and next-step planning.
5. Selected projects written at a business abstraction level.
6. Writing hub linking to Substack and LinkedIn.
7. Technical proof linking to GitHub without making code the main story.
8. Contact section connected to HiQ and customer conversations.

## Project Story Abstraction

Project examples should be written as:

- Business problem
- What I did
- Business value
- Role
- Technologies, kept secondary

This keeps the page readable for business leaders while preserving technical credibility.

## Visual Direction

The design should feel calm, credible, practical, and modern. It should use a restrained palette, clear information hierarchy, compact sections, and polished typography. It should not feel like a marketing splash page, a generic CV, or a one-note AI hype site.

## Technical Approach

Use a static site that can be opened directly in the browser:

- `index.html` for semantic content
- `styles.css` for responsive layout and visual design
- `script.js` for small progressive enhancements

No framework is required for the first version.

## Known User-Provided Content Needed Later

The first implementation can include editable placeholders for:

- Real Substack URL
- Real LinkedIn URL
- Real GitHub URL
- Real contact method
- Specific project names and customer-safe descriptions

The page should still look complete and useful before those links are replaced.
